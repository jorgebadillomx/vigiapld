# LLM de redacción del dictamen: Claude Sonnet 4.6 en AWS Bedrock

La capa de orquestación que redacta el dictamen usa **Claude Sonnet 4.6 a través de AWS Bedrock**, no DeepSeek (que proponía el brief). Se invoca con el cliente Bedrock de Anthropic (`AnthropicBedrockMantleClient`, paquete `Anthropic.Bedrock`, coherente con el stack ASP.NET Core) contra el endpoint Messages de Bedrock; el id de modelo en Bedrock lleva prefijo `anthropic.` (`anthropic.claude-sonnet-4-6`) y la región es requerida. Un solo modelo: como las reglas deciden y el LLM solo redacta (ADR-0004), no se usa el esquema de dos modelos "orquestador + razonador" del brief.

**Por qué:** es un producto de cumplimiento PLD con datos sensibles (padrones, LPB confidencial, expediente de 10 años). Bedrock da manejo de datos y residencia bajo el ecosistema AWS (IAM, regiones, contratos), lo que es más defendible ante una auditoría que una API LLM de terceros genérica. Claude Sonnet ofrece la calidad de redacción en español que el dictamen necesita. El motor ya aísla el LLM tras un puerto, así que el proveedor es intercambiable sin tocar el núcleo.

**Consecuencia:** lock-in con AWS Bedrock para inferencia. Sonnet 4.6 (~$3/$15 por 1M tokens input/output; verificar pricing regional de Bedrock) es más caro que DeepSeek, pero el costo de IA por corrida sigue muy por debajo de OpenSanctions (ver Further Notes del PRD) y se mitiga con caché de prompt (manual) de Bedrock y plantillas para los `descartado-por-nombre`. Algunas features de la API de Anthropic no están en Bedrock (caché automática, batches, Files API), pero la redacción del dictamen solo necesita el Messages API, que sí está soportado.
