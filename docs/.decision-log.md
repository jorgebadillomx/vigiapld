## 2026-06-24 — Revisión de costos y variante de hosting lean/VPS (input del propietario)

Disparador: el propietario tiene un **VPS compartido entre varios proyectos** y prioriza el costo más bajo dada la incertidumbre de volumen ("no sabemos si habrá muchos clientes").

### Decisiones de hosting

- **Variante lean/VPS compartido (escenario C) adoptada como opción de costo**, coexistiendo con el escenario "todo-AWS" (referencia de cumplimiento). Documentada en `architecture.md` §"Variante de despliegue: lean / VPS compartido". Costo marginal de este proyecto en VPS compartido: **~$5–10/mes** (vs ~$43–60 todo-AWS, ~$22–28 VPS dedicado).
- **Composición C:** cómputo + Postgres/Marten en el VPS (Coolify/Dokploy, DB con credenciales aisladas); **expediente WORM → S3 Object Lock en `mx-central-1`** (innegociable, no en disco de VPS — R3/NFR-4); backups off-box (pgBackRest → S3/B2); Bedrock y OpenSanctions por API.
- **Trampas de costo a evitar en cualquier escenario AWS:** NAT Gateway (usar VPC Gateway Endpoint S3, gratis); Secrets Manager → **SSM Parameter Store** (~$0); Graviton/ARM; diferir Fargate, ALB y RDS Multi-AZ hasta que el volumen lo pida.
- **Corrección a ADR-0007 (pendiente de aplicar):** Bedrock/Claude **no está en `mx-central-1`** (corre cross-region a EE.UU.); el argumento de "residencia de datos en AWS" aplica a almacenamiento (S3/RDS), no a la inferencia. Verificar en consola y matizar el ADR.

### Decisiones OpenSanctions

- **API-first.** La API hospedada (`/match`, €0.10/llamada exitosa, descuento >20k/mes) da **toda la funcionalidad de matching** (mismo motor `yente`, mismos datasets sanciones/PEP). Self-host (licencia bulk anual) se difiere hasta que `€0.10 × volumen > licencia anual`; el puerto `IOpenSanctionsPort` (US-56) permite el cambio sin tocar el núcleo.
- **R2 en modo API:** sellar en cada expediente la **versión de dataset que la API devuelve** (la API no permite pinear el matcher; equivale al pineo del self-host).
- **LFPDPPP en modo API:** el dato del sujeto (nombre+RFC+fecha) viaja a OpenSanctions (EU); base de licitud AML documentada. Self-host lo elimina si la residencia se vuelve requisito.

### AS-1 reabierto y precisado (bloqueador comercial, no de build)

El producto **distribuye datos de OpenSanctions a sus clientes** → requiere **licencia Reseller/OEM** (la más cara), y aplica **aunque se use la API**, no solo en self-host. OpenSanctions no publica montos.
**Acción antes de comercializar el canal white-label:** confirmar por escrito con ventas de OpenSanctions (a) si la API metered permite mostrar resultados a clientes en un SaaS y (b) el costo de la licencia reseller. Esto gatea el costo variable y el margen (SM-4 / AS-6).
